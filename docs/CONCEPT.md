# Family OS (Prev. Print2Cal) - Concept & Vision

> **Version:** 7.0 (Ethical Infrastructure Edition)
> **Last Updated:** 2026/02/07
> **Concept:** **「生活のナレッジ（知識）を資産化する」**

---

## 1. ビジョン (Vision)
**「親の脳内メモリを解放し、家族に『安心』をインストールする」**

私たちは単なる「プリントOCRツール」を作るのではない。
「明日の準備」という短期的なタスク処理だけでなく、「去年の運動会、何が必要だった？」という**生活の運用知識（ナレッジ）** を蓄積し、再利用可能にする。
**「家族のためのGithub / Notion」** を構築する。

## 2. 課題定義 (The Problem)

### 表面的な課題
* 学校・園からのプリント管理が限界。
* Googleカレンダーへの手入力が面倒で、後回しになりがち。

### 真の課題 (Pain Point)
* **情報の非対称性**: 「パパは知らない（言わないと伝わらない）」ことによる確認コストと摩擦。
* **ナレッジの消失**: 毎年同じ行事があるのに、毎回ゼロから「何が必要だっけ？」と悩み、同じミスを繰り返す徒労感。
* **人質への恐怖**: 便利なツールを使いたいが、解約したらデータが消える恐怖で導入に踏み切れない。

## 3. ソリューション (The Solution)
**Core Product: Family OS (Parent OS)**

1.  **Input (Frictionless)**
    * 写真を撮って送るだけ。誰が送ってもいい。
2.  **Process (Recoverable AI)**
    * **思想:** 「AIは間違える。だから直しやすくした。」(Undo/Rescue)
    * **機能:** 一括削除、誤除外の救出。
3.  **Output (Knowledge Base)**
    * **フロー（通知）:** 明日のリマインド。
    * **ストック（資産）:**
        * 「去年の持ち物リスト」
        * 「提出物の書き方見本」
        * 「先生からの注意事項ログ」

## 4. 提供価値とマネタイズ (Value & Business)
**「生活ナレッジの運用（Knowledge Ops）」に課金する**
データは人質に取らない。**「データを活かす機能」** に対価を支払ってもらう。

| プラン | スタンダード (Free) | プレミアム (Pro) |
| :--- | :--- | :--- |
| **コンセプト** | **「便利なタスク処理ツール」** | **「家族のナレッジベース」** |
| **データ保存** | **無期限 (消さない)** | **無期限 (消さない)** |
| **過去検索** | 直近 3ヶ月のみ (閲覧は可能) | **全期間検索 & AI活用** |
| **ナレッジ活用** | なし | **「去年のこれ、どうしたっけ？」**<br>AIによる過去ログ参照・提案 |
| **解約時の挙動** | - | **データは凍結 (Freeze)**<br>消えないが、検索・AI活用が停止する |
| **価格** | ¥0 | **月額 ¥300〜¥500** |

**勝算のロジック (The Netflix Model):**
* 解約してもデータは消さない。「いつでも戻ってこれる」安心感が、逆に継続率を高める。
* 価値は「思い出」だけでなく**「来年の自分への申し送り事項」**。
    * 例: 「来年の遠足は、レジャーシートを大きめにすること」というメモが、翌年リマインドされる価値。

## 5. 指標設計 (KPIs)

### North Star Metric (思想)
🌟 **Knowledge Reuse Rate (ナレッジ再利用率)**
* 定義: 過去のログ（イベント・プリント）が検索・参照され、今のタスク解決に役立った回数。

### Practical KPIs (実務・行動指標)
1.  **3ヶ月超データ率**: 3ヶ月以上前のデータを検索・閲覧しているユーザーの割合（＝ストック価値を感じている層）。
2.  **検索→課金転換率**: 「過去ログ検索」を実行しようとして、Proプランに加入した率（＝ナレッジが必要になった瞬間）。
3.  **家庭あたり保存枚数**: データ密度＝解約耐性。
4.  **Unforgotten Rate**: 通知によってトラブルを回避できた推定回数。

## 6. ロードマップ (Roadmap)

### Phase 1-2: 信頼性の確立 (✅ Completed)
* 基礎機能の実装（OCR、フィルタリング、セキュリティ）。

### Phase 3: 回復性の担保 (Recoverability) 👈 Current Focus
* **ゴール:** 「間違っても大丈夫」な体験。
* **実装:** Flex Message化、Undo機能、救出機能。

### Phase 4: 共有とインフラ化 (Sharing & Infrastructure)
* **ゴール:** 「ママツール」から「家族インフラ」へ。
* **実装:** グループLINE対応（共有無料開放）。

### Phase 5: 資産化と収益化 (Asset & Monetization)
* **ゴール:** 「生活ナレッジ」の構築。
* **実装:**
    * 検索機能の実装。
    * 「凍結（Freeze）」ロジックの実装（削除ではない）。
    * Stripe決済。

## 7. 開発マニフェスト (Development Philosophy)
1.  **Don't hold data hostage:** データはユーザーのもの。解約しても消すな。凍結せよ。
2.  **Build a Knowledge Base:** 単なる「タスク消化」ではなく「来年の楽」を作れ。
3.  **Be a Niche Infrastructure:** ユニコーンを目指すな。家族のライフラインになれ。
